项目2 创建数据库

2.1项目目标

2.1.1了解数据库组成

数据库是表（Table）、视图（Index）、存储过程（Store Procedure）、触发器（Trigger）等数据库对象的集合，是数据库管理系统的核心内容。数据库的存储机构分为逻辑存储结构和物理存储结构。

数据库逻辑存储结构 数据库对象 表、索引、触发器、存储过程、约束和用户等
数据库物理存储结构 数据库文件 数据库文件和事务日志文件

其中，数据库文件分为

主数据库文件（.mdf） 用来存储数据库的启动信息和部分或全部数据 一个数据库有且只能有一个主数据库文件
次数据库文件（.ndf） 包含除主数据库文件外的所有数据库文件 如果主数据库文件足够大到容纳数据库中所有数据时，数据库不一定需要次数据库文件

​

事务日志文件（.ldf） 用来记录在数据库中发生的所有修改和导致这些修改的事务，保存所有可能用来恢复数据库的事务信息， 每个数据库至少有一个事务日志文件，最小为512KB，理想容量为数据库大小的25%~40%

2.1.2理解数据库文件、文件组的种类和作用

1.数据库文件

主数据库文件（.mdf） 用来存储数据库的启动信息和部分或全部数据 一个数据库有且只能有一个主数据库文件
次数据库文件（.ndf） 包含除主数据库文件外的所有数据库文件 如果主数据库文件足够大到容纳数据库中所有数据时，数据库不一定需要次数据库文件

​

事务日志文件（.ldf） 用来记录在数据库中发生的所有修改和导致这些修改的事务，保存所有可能用来恢复数据库的事务信息， 每个数据库至少有一个事务日志文件，最小为512KB，理想容量为数据库大小的25%~40%

​ 2.文件组

​ 是文件的逻辑集合，用来对文件进行分组，类似于文件夹。

​ 事务日志文件不存在于任一文件组。

主文件组（PRIMARY） 包含主数据文件和没有指定文件组的次数据文件
用户自定义文件组 通常在大型数据库应用系统中使用，由用户自行创建，通过在不同磁盘上创建文件组的方法来提高查询效率

2.1.3掌握在管理数据库时常用的系统存储过程

管理常用数据库的系统存储过程有以下几个：

系统存储过程sp_helpdb：查看数据库的拥有者、数据库容量、创建日期及状态等数据库的基本信息。

系统存储过程sp_helpfilegroup：查看数据库中文组件信息。

系统存储过程sp_rename：重命名数据库。

2.1.4会创建数据库

2.1.4.1使用SSMS创建图书借阅数据库JY
  1. 启动“SSMS”，连接到“本地默认实例”→在“对象资源管理器”窗口中右击“数据库”→“新建数据库”。
  2. 输入“数据库名称”→确定主数据库文件和事务日志文件的“初始大小”→设定”增长方式“→更改数据库文件“存储路径”。
  3. 点击“确定“。

​

2.1.4.2使用CREATE DATABASE语句创建数据库

1.启动SSMS，连接到“本地默认实例”→单击”新建查询“按钮→在“查询编辑器”窗口中输入创建数据库的SQL语句如下：

ON
PRIMARY
(
    NAME = JY1,
    FILENAME = 'e:\JY.mdf',
    SIZE = 5MB,
    MAXSIZE = 40MB
)
LOG ON
(
    NAME = JY1_log,
    FILENAME = 'e:\JY.ldf',
    SIZE = 15MB,
    MAXSIZE = 60MB,
    FILEGROWTH = 6MB
)
GO

2.单击”执行“命令（F5），即可成功创建数据库JY。

由于创建数据库后，”对象资源管理器“不会自动刷新，所以必须手动刷新才能看见已经创建好的数据库。

2.1.5会管理数据库，包括修改数据库设置、重命名和删除数据库

2.1.5.1修改JY数据库设置
使用SSMS修改数据库JY的设置，缩减数据库容量

1.打开“SSMS”→”本地默认实例“→”对象资源管理器“→右击需要修改信息的”JY“数据库→“属性”→”数据库属性“→根据实际需要进行设置

2.打开“SSMS”→”本地默认实例“→”对象资源管理器“→右击需要缩减容量的”JY“数据库→“任务”→“收缩”→“数据库”→“确定”

使用ALTER DATABASE语句修改JY的设置

1.在数据库中添加文件组：打开“SSMS”→”本地默认实例“→在“查询编辑器”窗口中输入新增名为group1文件组的ADD FILEGROUP语句：

ALTER DATABASE JY
ADD FILEGROUP group1    --新增文件组
GO
sp_helpfilegroup    --查看数据库中文件组信息
GO

​

2.扩充数据库的容量——修改数据文件和事务日志文件容量：在“查询编辑器”窗口中输入修改现有的数据文件和事务日志文件容量的MODIFY FILE语句，如下

ALTER DATABASE JY
MODIFY FILE
(
    NAME = JY,
      SIZE = 30MB
)
GO
ALTER DATABASE JY
MODIFY FILE
(
    NAME = JY_log,
      SIZE = 40MB
)
GO
SP_helpdb JY    --查看数据库JY扩容后的信息
GO

​

3.扩充数据库的容量——新增次数据文件和事务日志文件：在“查询编辑器”窗口中输入新增次数据文件和事务日志文件的ADD FILE语句，如下

--新增一个次数据文件
ALTER DATABASE JY
ADD FILE
(
    NAME = 'JY_data1',
      FILENAME = 'e:\JY_data1.ndf',
      SIZE = 5MB,
      MAXSIZE = 20MB,
      FILEGROWTH = 3MB
)
GO
--新增一个事务日志文件
ALTER DATABASE JY
ADD LOG FILE
(
    NAME = 'JY_log1',
      FILENAME = 'e:\JY_log1.ldf',
      SIZE = 5MB,
      MAXSIZE = 20MB,
      FILEGROWTH = 3MB
)
GO
SP_helpdb JY    --查看数据库JY扩容后的信息
GO

​

4.缩减数据库的容量——删除数据文件或事务日志文件：在“查询编辑器”窗口中输入删除指定的数据文件或事务日志文件的REMOVE FILE语句，如下

ALTER DATABASE JY
REMOVE FILE JY_data1
GO
ALTER DATABASE JY
REMOVE FILE JY_log1
GO
sp_helpdb JY    --查询数据库JY减容后的信息
GO

​

5.缩减数据库容量——缩减数据文件或事务日志文件的容量：在“查询编辑器”窗口中输入收缩数据库的数据文件或事务日志文件的DBCC SHRINKFILE的语句，如下

USE JY
GO
DBCC SHRINKFILE(JY,5)
GO
sp_helpdb JY    --查看数据库JY的信息
GO
2.1.5.2重命名和删除数据库JY
重命名数据库JY

重命名数据库有以下的要求：只有数据库管理员可以重命名数据库；其他用户已断开与数据库的连接；将数据库的选项修改为单用户模式。

1.将数据库的状态修改为单用户模式

右击”JY”→“属性”→“选项”→“状态”→“限制访问”→“SINGLE_USER”

2.重命名数据库

方式1：右击“JY”→“重命名”

方式2：“查询编辑器”→使用系统存储过程sp_rename重命名数据库

sp_renamedb    'JY','JY1'
GO
删除数据库JY

方式1：使用SSMS删除数据库

​ 右击“JY”→“删除”

方式2：使用DROP DATABASE语句删除数据库

DROP DATABASE JY
GO

2.1.6会分离和附加数据库

分离数据库JY

要求：限制数据库为单用户模式

1.右击”JY“→“任务”→“分离”→”确定“

附加数据库JY

1.右击“数据库”→“附加”→“附加数据库”→”添加“→找到JY.mdf文件所在目录→选择要附加的数据库文件JY.mdf→”确定“

2.2项目小结

1.SQL Server的5个系统数据库：master、model、tempdb、msdb、resource。

2.三种数据库文件：主数据文件、次数据文件、事务日志文件。

3.数据库默认的主文件组PRIMARY。

4.创建数据库实际上是创建数据文件和事务日志文件，以及设置它们的属性和彼此之间的关系。

5.在本地客户端使用SSMS会比较方便，在服务器端使用Transact-SQL会比较迅速，所以要根据实际情况选择操作方法。

6.在配置数据库之前要大致明白每个配置选项的作用，创建数据库和配置数据库的重点是文件和文件组，以及自动增长的属性设置。

7.分离暂时不用的数据库可以减轻SQL Server 2012的负担。

8.数据库使用一段时间后可能会变得很大，可以通过收缩数据库的方式来释放空间。

9.对于不需要的数据库，可以删除。
